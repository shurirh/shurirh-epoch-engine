# Epoch Engine — Specification v0.3

## Objetivo

La versión **v0.3** introduce mejoras estructurales y funcionales al motor temporal:

1. Reestructuración del proyecto en capas claras.
2. Sistema completo de **importación y exportación JSON**.
3. Primer **renderer visual del grafo temporal**.
4. Preparación para integración con datos externos (CSV/XLS).

Esta versión **no incluye aún el editor visual**, que se planifica para **v0.4**.

---

# 1. Nueva estructura del proyecto

El proyecto se reorganiza en módulos claros.

```
epoch-engine/
│
├─ core/
│   ├─ graph.js
│   ├─ node.js
│   ├─ edge.js
│   ├─ suceso.js
│   └─ epochEngine.js
│
├─ io/
│   ├─ jsonImporter.js
│   └─ jsonExporter.js
│
├─ ui/
│   ├─ renderer.js
│   └─ styles.css
│
├─ utils/
    └─ csvAdapter.js
```

## Principio de diseño

Cada capa tiene responsabilidades claras:

### core

Motor puro de datos.

No contiene:

* DOM
* HTML
* CSS
* renderizado

Debe funcionar en:

* Node.js
* navegador
* entornos server

---

### io

Responsable de **serialización y deserialización de datos**.

Incluye:

* importadores
* exportadores
* adaptadores de formatos

---

### ui

Visualización del grafo temporal.

No contiene lógica de negocio.

---

### utils

Herramientas auxiliares para conversión de datos.

---

# 2. Mejoras en EpochEngine

## 2.1 Validación de tipos de relación

El método `relate()` debe validar que el tipo pertenezca a `EdgeTypes`.

Ejemplo:

```
if (!Object.values(EdgeTypes).includes(type)) {
  throw new Error(`Tipo de relación inválido: ${type}`);
}
```
NOTA: ya lo he actualizado manualmente, por favor revisarlo!

---

## 2.2 Generación de ID de aristas

Se reemplaza el uso de `Date.now()` por:

```
crypto.randomUUID()
```

Ejemplo:

```
const edgeId = crypto.randomUUID();
```

Debe funcionar tanto en:

* Node.js
* navegadores modernos

NOTA: ya lo he actualizado el codigo manualmente, por favor revisarlo!

---

## 2.3 Nuevo método `exportJSON()`

El motor debe poder exportar su estado completo.

```
exportJSON()
```

Devuelve:

```
{
  events: [],
  relations: []
}
```

Formato esperado:

```
{
  "events": [
    {
      "id": "e1",
      "title": "Descubrimiento de Marte",
      "description": "...",
      "date": 2032,
      "tags": ["space"],
      "meta": {}
    }
  ],
  "relations": [
    {
      "id": "r1",
      "from": "e1",
      "to": "e2",
      "type": "causes",
      "metadata": {}
    }
  ]
}
```

---

## 2.4 importJSON mejora

`importJSON()` debe devolver un reporte:

```
{
  eventsImported: number,
  relationsImported: number
}
```

También debe validar:

* IDs duplicados
* tipos de relación inválidos

# 3. Sistema de renderizado (ui/renderer.js)

Se introduce el primer renderer visual.

## Objetivo

Permitir visualizar el grafo temporal en un navegador.

---

## Estrategia técnica

El renderer utilizará **SVG**.

Razones:

* escalable
* fácil interacción
* compatible con DOM

---

## API del renderer

```
renderGraph(engine, container)
```

Parámetros:

```
engine      → instancia de EpochEngine
container   → elemento DOM donde renderizar
```

Ejemplo:

```
const engine = new EpochEngine()

renderer.renderGraph(engine, document.getElementById("timeline"))
```

---

## Representación visual

### Sucesos

Se representan como:

```
SVG Rectangle
```

con etiqueta de texto.

---

### Relaciones

Se representan como:

```
SVG path
```

con flecha direccional.

---

### Colores sugeridos

```
precedes → gris
causes   → azul
branches → naranja
merges   → verde
loops    → rojo
```

---

## Layout inicial

El layout inicial puede ser simple:

```
grid layout
```

Los nodos se colocan automáticamente con separación uniforme.

En versiones futuras se puede implementar:

* layout dirigido por fuerza
* layout temporal

# 4. Adaptador CSV/XLS

Se introduce un sistema simple para convertir planillas en JSON compatible.

Archivo:

```
utils/csvAdapter.js
```

---

## Estructura esperada de la planilla

Hoja 1: **timeline**

| id | title | description | start | end | tags | meta | precedes | causes | branches | merges | loops |
| -- | ----- | ----------- | ----- | --- | -------- | ---- | ---- | -------- | ------ | -------- | ------ | ------- |

Ejemplos:

```
e1,First Landing,,2032-04-10,,space|mars,{"location":"Mars"},e2,,,,
e2,First walk on Mars,,2032-04-10,,space|mars,{"location":"Mars"},,e1,,,
e3,Temporal divergence,,2032-04-11,,timeline,{"reason":"paradox"},,,e4,,
e4,Alternate future,,2032-04-11,,altfuture,{},,,,e2|e3,
```

---

## Función del adaptador

```
csvToJSON(csvTimeline)
```

Devuelve:

```
{
  events: [],
  relations: []
}
```

compatible con `importJSON()`.

---

# 5. Ejemplo de flujo completo

```
CSV / XLS
   ↓
csvAdapter
   ↓
JSON
   ↓
EpochEngine.importJSON()
   ↓
Graph
   ↓
Renderer
```
